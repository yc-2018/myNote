## ==13.== shell 脚本怎么接收参数

> <kbd>2024.04.15</kbd> <kbd>GPT-4 Turbo + You.com AI</kbd>
>
> ```sh
> #!/bin/bash
> echo "第1个参数: $1"
> echo "第2个参数: $2"
> ```
>
> 执行
>
> ```sh
> bash example.sh 参数1 参数2
> ```
>
> ---
>
> **没传也有默认值**
>
> 在Shell脚本中接收参数时，如果想要为某些参数设置默认值（即在调用脚本时没有传递这些参数），可以使用以下方法：
>
> ```bash
> #!/bin/bash
>
> # 使用${1:-默认值}的语法来为第一个参数设置默认值
> param1=${1:-"默认值1"}
>
> # 使用${2:-默认值}的语法来为第二个参数设置默认值
> param2=${2:-"默认值2"}
>
> echo "第一个参数: $param1"
> echo "第二个参数: $param2"
> ```
>
> 这个脚本中，`${1:-"默认值1"}` 表示如果第一个参数没有被传递，那么`param1`将使用`"默认值1"`作为其值。同理，`${2:-"默认值2"}` 对第二个参数做了同样的处理。
>
> 例如，如果你运行脚本但不传递任何参数：
>
> ```bash
> bash example.sh
> ```
>
> 输出将会是：
>
> ```
> 第一个参数: 默认值1
> 第二个参数: 默认值2
> ```
>
> 如果你传递了一些参数，比如：
>
> ```bash
> bash example.sh 实际值1
> ```
>
> 输出将会是：
>
> ```
> 第一个参数: 实际值1
> 第二个参数: 默认值2
> ```
>
> 这种方法可以灵活地为脚本中的参数提供默认值，确保即使在没有提供必要的参数时，脚本也能正常运行。
