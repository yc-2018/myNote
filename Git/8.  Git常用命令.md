## ==8.== Git常用命令

> #### Git最常用的20条命令
>
> 1. `git init`：初始化一个新的Git仓库。 例如：`git init`
> 2. `git clone [url]`：克隆一个仓库到本地。 例如：`git clone https://github.com/example/repo.git`
> 3. `git add [file]`：添加文件到暂存区。 例如：`git add example.txt`
> 4. `git commit -m "[message]"`：提交更新到仓库，并附加提交信息。 例如：`git commit -m "Initial commit"`
> 5. `git status`：查看仓库当前状态。 例如：`git status`
> 6. `git push [alias] [branch]`：将本地仓库的更改推送到远程仓库。 例如：`git push origin master`
> 7. `git pull [alias] [branch]`：从远程仓库拉取最新版本并合并到本地。 例如：`git pull origin master`
> 8. `git branch`：列出、创建或删除分支。 例如：`git branch -a`
> 9. `git checkout [branch-name]`：切换到指定分支。 例如：`git checkout develop`
> 10. `git merge [branch]`：合并指定分支到当前分支。 例如：`git merge feature/new-feature`
> 11. `git rebase [branch]`：变基指定分支到当前分支。 例如：`git rebase master`
> 12. `git diff`：查看未暂存的更改。 例如：`git diff`
> 13. `git reset [file]`：取消暂存的文件。 例如：`git reset example.txt`
> 14. `git log`：查看提交历史。 例如：`git log`
> 15. `git show [commit]`：显示某次提交的内容。 例如：`git show 1a2b3c4d`
> 16. `git rm [file]`：从仓库中删除文件。 例如：`git rm deleted_file.txt`
> 17. `git stash`：暂存未提交的更改。 例如：`git stash`
> 18. `git tag [tag-name]`：标记特定的提交。 例如：`git tag v1.0.0`
> 19. `git fetch [alias]`：从远程仓库获取数据到本地。 例如：`git fetch origin`
> 20. `git remote add [alias] [url]`：添加一个新的远程仓库。 例如：`git remote add origin https://github.com/example/repo.git`
>
> Git 提供了更多的命令和选项来处理复杂的版本控制需求。这里是一些额外的 Git 命令，这些命令也非常实用，尤其在处理大型项目和复杂的协作场景时：
>
> 21. `git config`：配置Git的参数，如用户信息等。
>     例如：`git config --global user.name "Your Name"`
>
> 22. `git cherry-pick [commit]`：选择一个提交从一个分支并应用它到当前分支。
>     例如：`git cherry-pick 4a5b6c7d`
>
> 23. `git revert [commit]`：撤销指定的提交，并创建一个新的提交。
>     例如：`git revert 1a2b3c4d`
>
> 24. `git rebase -i [base]`：交互式变基，可以选择性地修改一系列提交。
>     例如：`git rebase -i HEAD~5`
>
> 25. `git stash pop`：应用被暂存的更改并从暂存列表中删除。
>     例如：`git stash pop`
>
> 26. `git stash apply`：应用被暂存的更改但不从暂存列表中删除。
>     例如：`git stash apply`
>
> 27. `git stash list`：查看所有暂存的更改列表。
>     例如：`git stash list`
>
> 28. `git stash clear`：清除所有暂存的更改。
>     例如：`git stash clear`
>
> 29. `git tag -a [tag-name] -m "[message]"`：创建一个带有注释的标签。
>     例如：`git tag -a v1.1 -m "Version 1.1 released"`
>
> 30. `git log --graph`：以图形方式显示提交历史。
>     例如：`git log --graph`
>
> 31. `git blame [file]`：查看文件的行修改记录。
>     例如：`git blame README.md`
>
> 32. `git remote show [alias]`：显示一个远程仓库的详细信息。
>     例如：`git remote show origin`
>
> 33. `git remote rename [old-name] [new-name]`：重命名一个远程仓库。
>     例如：`git remote rename origin upstream`
>
> 34. `git push --tags`：推送所有标签到远程仓库。
>     例如：`git push --tags`
>
> 35. `git checkout -- [file]`：放弃对文件的修改。
>     例如：`git checkout -- example.txt`
>
> 36. `git clean -f`：删除未跟踪的文件。
>     例如：`git clean -f`
>
> 37. `git branch -d [branch-name]`：删除一个本地分支。
>     例如：`git branch -d feature-branch`
>
> 38. `git branch -D [branch-name]`：强制删除一个本地分支。
>     例如：`git branch -D feature-branch`
>
> 39. `git branch -m [old-branch-name] [new-branch-name]`：重命名本地分支。
>     例如：`git branch -m old-name new-name`
>
> 40. `git push origin --delete [branch-name]`：删除远程分支。
>     例如：`git push origin --delete feature-branch`
>
> 这些命令进一步扩展了Git的功能，使用户能够更细致地管理代码库的版本控制和历史记录。
